
What is Virtualization technology in the context of cloud computing, and what are its main advantages?	"<b>English:</b> It is the technology that divides a physical server into multiple independent virtual servers.<br>• <i>Efficiency:</i> Better resource utilization.<br>• <i>Savings:</i> Fewer physical machines mean lower costs.<br>• <i>Practicality:</i> The provider handles all heavy hardware maintenance.<br><br><b>Portuguese:</b> É a tecnologia que divide um servidor físico em vários servidores virtuais independentes.<br>• <i>Eficiência:</i> Melhor uso dos recursos.<br>• <i>Economia:</i> Menos máquinas físicas significam custos menores.<br>• <i>Praticidade:</i> O provedor cuida de toda a manutenção pesada de hardware."
What is the conceptual difference between Vertical Scalability and Horizontal Scalability?	"<b>English:</b><br>• <i>Vertical Scalability:</i> Upgrades the capacity of the current machine (adding more RAM or CPU).<br>• <i>Horizontal Scalability:</i> Adds more machines/servers to distribute the workload.<br><br><b>Portuguese:</b><br>• <i>Escalabilidade Vertical:</i> Dá um ""upgrade"" na máquina atual (adicionando mais RAM ou CPU).<br>• <i>Escalabilidade Horizontal:</i> Adiciona mais máquinas/servidores para dividir o trabalho."
What are the key differences in responsibilities and target audiences for IaaS, PaaS, and SaaS?	"<b>English:</b><br>• <i>IaaS (Infrastructure):</i> Provides raw hardware; the user (SysAdmin) manages the OS and apps.<br>• <i>PaaS (Platform):</i> Provides hardware + OS/middleware; the developer only manages the code.<br>• <i>SaaS (Software):</i> A ready-to-use application hosted on the internet for end-users.<br><br><b>Portuguese:</b><br>• <i>IaaS (Infraestrutura):</i> Oferece hardware básico bruto; o usuário (SysAdmin) gerencia o S.O. e os aplicativos.<br>• <i>PaaS (Plataforma):</i> Oferece hardware + S.O./middleware; o desenvolvedor apenas gerencia o código.<br>• <i>SaaS (Software):</i> Um aplicativo pronto para uso hospedado na internet para usuários finais."
What is FaaS (Function as a Service / Serverless) and how does its billing model work?	"<b>English:</b> It is a model where you do not manage servers; instead, you only run specific pieces of code (functions) triggered by events.<br>• <i>Billing:</i> You only pay for the exact milliseconds the code actually runs.<br><br><b>Portuguese:</b> É um modelo onde você não gerencia servidores; em vez disso, você apenas executa pedaços específicos de código (funções) ativados por eventos.<br>• <i>Cobrança:</i> Você só paga pelos milissegundos exatos em que o código realmente roda."
What is the difference between a Hybrid Cloud and a Multicloud strategy?	"<b>English:</b><br>• <i>Hybrid Cloud:</i> Combines different deployment models (usually Private + Public) that share data.<br>• <i>Multicloud:</i> Combines different public cloud providers (e.g., AWS + GCP + Azure) to avoid vendor lock-in.<br><br><b>Portuguese:</b><br>• <i>Nuvem Híbrida:</i> Combina diferentes modelos de implantação (geralmente Privada + Pública) que compartilham dados.<br>• <i>Multicloud:</i> Combina diferentes provedores de nuvem pública (ex: AWS + GCP + Azure) para evitar a dependência de fornecedor."
What is the concept of Cloud Bursting?	"<b>English:</b> It is a configuration where a private cloud automatically redirects its excess traffic spikes to a public cloud when its local capacity limit is reached.<br><br><b>Portuguese:</b> É uma configuração onde uma nuvem privada redireciona automaticamente seus picos de excesso de tráfego para uma nuvem pública quando seu limite de capacidade local é atingido."
What is a Community Cloud?	"<b>English:</b> An infrastructure shared exclusively by a group of organizations that have common interests, missions, or strict security requirements.<br><br><b>Portuguese:</b> Uma infraestrutura compartilhada exclusivamente por um grupo de organizações que possuem interesses, missões ou requisitos rígidos de segurança em comum."
What is the GDPR rule regarding the transfer of personal data (PII) of EU citizens outside its geographical borders?	"<b>English:</b> Personal data cannot leave the borders of the European Union unless the destination country/entity guarantees the same level of strict legal protection required by EU law.<br><br><b>Portuguese:</b> Dados pessoais não podem sair das fronteiras da União Europeia, a menos que o país/entidade de destino garanta o mesmo nível de proteção jurídica rigorosa exigido pela lei da UE."
How do Data Scientists, Machine Learning Scientists, Data Engineers, and Data Analysts utilize cloud infrastructure?	"<b>English:</b><br>• <i>Data Scientist:</i> Runs heavy analyses quickly and cheaply under a pay-as-you-go model.<br>• <i>Machine Learning Scientist:</i> Accesses specialized hardware like GPUs to train Deep Learning models.<br>• <i>Data Engineer:</i> Builds pipelines to extract, transform, and store Big Data.<br>• <i>Data Analyst:</i> Connects to the cloud using Business Intelligence (BI) tools to consume data and generate reports.<br><br><b>Portuguese:</b><br>• <i>Cientista de Dados:</i> Executa análises pesadas de forma rápida e barata sob o modelo pague-pelo-que-usar.<br>• <i>Cientista de Machine Learning:</i> Acessa hardware especializado como GPUs para treinar modelos de Deep Learning.<br>• <i>Engenheiro de Dados:</i> Constrói pipelines para extrair, transformar e armazenar Big Data.<br>• <i>Analista de Dados:</i> Conecta-se à nuvem usando ferramentas de Business Intelligence (BI) para consumir dados e gerar relatórios."
What are the equivalents for raw Object Storage, Virtual Machines, and Managed Relational Databases in AWS, Azure, and Google Cloud?	"<b>English:</b><br>• <i>Object Storage:</i> AWS <b>S3</b> | Azure <b>Blob Storage</b> | Google <b>Cloud Storage</b>.<br>• <i>Virtual Machines:</i> AWS <b>EC2</b> | Azure <b>Virtual Machines</b> | Google <b>Compute Engine</b>.<br>• <i>Relational Databases:</i> AWS <b>RDS</b> | Azure <b>SQL Database</b> | Google <b>Cloud SQL</b>.<br><br><b>Portuguese:</b><br>• <i>Armazenamento de Objetos:</i> AWS <b>S3</b> | Azure <b>Blob Storage</b> | Google <b>Cloud Storage</b>.<br>• <i>Máquinas Virtuais:</i> AWS <b>EC2</b> | Azure <b>Virtual Machines</b> | Google <b>Compute Engine</b>.<br>• <i>Bancos de Dados Relacionais:</i> AWS <b>RDS</b> | Azure <b>SQL Database</b> | Google <b>Cloud SQL</b>."
What are the equivalents for Data Warehouses, Real-time Stream Analytics, and Machine Learning platforms in AWS, Azure, and Google Cloud?	"<b>English:</b><br>• <i>Data Warehouse:</i> AWS <b>Redshift</b> | Google <b>BigQuery</b>.<br>• <i>Stream Analytics:</i> AWS <b>Kinesis</b> | Azure <b>Stream Analytics</b> | Google <b>Dataflow</b>.<br>• <i>ML Platforms:</i> AWS <b>SageMaker</b> | Azure <b>Machine Learning</b> | Google <b>AutoML</b>.<br><br><b>Portuguese:</b><br>• <i>Data Warehouse:</i> AWS <b>Redshift</b> | Google <b>BigQuery</b>.<br>• <i>Análise de Streaming:</i> AWS <b>Kinesis</b> | Azure <b>Stream Analytics</b> | Google <b>Dataflow</b>.<br>• <i>Plataformas de ML:</i> AWS <b>SageMaker</b> | Azure <b>Machine Learning</b> | Google <b>AutoML</b>."
What is Google Cloud Anthos and what problem does it solve?	"<b>English:</b> It is a service that allows customers to manage and deploy workloads across hybrid and multi-cloud environments. It solves the problem of <b>vendor lock-in</b>.<br><br><b>Portuguese:</b> É um serviço que permite gerenciar e implantar cargas de trabalho em ambientes híbridos e multi-nuvem. Ele resolve o problema da <b>dependência de fornecedor (vendor lock-in)</b>."
